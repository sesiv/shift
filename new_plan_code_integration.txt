План внедрения кода с изменением исходного pooling-модуля multilingual-e5-large-instruct

1. Подготовить train, validation и test, а также автоматически собрать positive и negative пары из исходных классов.
Positive - две записи одного класса.
Negative - две записи разных классов.

2. Посчитать idf только по train-корпусу и сохранить словарь token -> idf или token_id -> idf.
Сразу определить, на каком уровне будут считаться веса: по словам до токенизации или по token id модели.

3. Установить проект и библиотеку в editable-режиме через pip install -e ., чтобы можно было править исходный модуль и сразу видеть изменения без переустановки.

4. Найти в используемом стеке не transformer-слой, а исходный код формирования sentence embedding.
Если проект работает через sentence-transformers, менять модуль Pooling.py.
Если проект использует чистый transformers, менять функцию average_pool после last_hidden_state.

5. Не делать внешнюю обертку, а изменить сам исходный pooling-модуль.
Стандартный mean pooling оставить как отдельный режим.
Добавить новый режим, например tfidf_weightedmean.

6. Внутри pooling-модуля заменить формулу агрегации.
Было:
v = sum(h_i) / n

Станет:
w_i = 1 + alpha * tfidf_i
v = sum(w_i * h_i) / sum(w_i)

7. Учесть attention_mask внутри исходного модуля.
PAD-токены не должны участвовать ни в сумме эмбеддингов, ни в сумме весов.
TF-IDF веса тоже должны маскироваться.

8. Оставить L2-нормализацию после pooling без изменений.

9. Добавить в новый pooling хотя бы один обучаемый параметр.
Минимум - alpha.
Если нужен более сильный вариант, добавить контекстную часть:
w_i = 1 + alpha * tfidf_i + beta * g_i

10. Заморозить весь encoder.
На первом этапе обучать только параметры нового pooling-модуля.

11. Дообучить модель на своем корпусе через contrastive или ranking loss.
Оптимизировать embedding space: запрос должен быть ближе к positive, чем к negative.

12. Сначала получить baseline на исходной модели со стандартным mean pooling.
Потом сравнить его с новой реализацией.

13. Провести минимум три сравнения:
baseline mean pooling,
TF-IDF pooling без дообучения,
TF-IDF pooling после дообучения.

14. Опционально провести второй этап эксперимента.
После оценки нового pooling разморозить только последний transformer block и проверить, даст ли это прирост.

15. Итоговая цель внедрения.
Изменить не внутренние слои xlm-roberta-large, а именно исходный модуль pooling, который формирует финальный sentence embedding из last_hidden_state.
